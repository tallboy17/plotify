Google Sign-In Test

This will test the complete Google OAuth flow:

  1. Sign in with Google (opens popup)
  2. Get Google ID token from the popup
  3. Send ID token to our backend for verification
  4. Backend checks centralized USERS_BIN for existing user
  5. If new user, creates user record in centralized USERS_BIN
  6. Backend generates JWT token and returns user info with subscription data
  7. Frontend stores JWT token and user data for API calls

Setup Instructions

Before testing, make sure you have:

  • Set GOOGLE_CLIENT_ID in your .env file
  • Set JSONBIN_API_KEY in your .env file
  • Set USERS_BIN in your .env file (JSONBin.io bin ID for user storage)
  • Installed firebase-admin package
  • Added serviceAccountKey.json to your project root
  • Configured Google OAuth in Google Cloud Console
  • Created a JSONBin.io bin for user storage and initialized it with {}

Manual Token Test

If you have a Google ID token, you can test it directly:

POST /api/auth/google
Content-Type: application/json

{
  "id_token": "your_google_id_token_here"
}
